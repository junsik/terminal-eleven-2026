# UI_FLOW.md — 화면 설계 및 UX 규칙(Flutter)

## 1. 최소 화면(MVP)
1) HomeScreen: 다음 경기, 상태(피로/신뢰/자신감), 인박스 알림
2) TrainingScreen: 오늘 선택(훈련/휴식/재활)
3) MatchScreen: 상황+선택지, 로그 피드, 진행(현재 하이라이트 n/12)
4) SummaryScreen: 최종 스코어, 평점, 신뢰/XP 변화, 다음 일정
5) CareerScreen: 스탯, 레벨, 기록(골/어시), 최근 평점 그래프(추후)
6) InboxScreen: 코치/에이전트/팬 메시지

## 2. MatchScreen UX 규칙
- 상단: 스코어, 분, 하이라이트 진행률
- 중앙: 현재 상황 카드(1~2문장)
- 하단: 선택지 버튼 2~4개(Primary/Secondary)
- 로그: 선택 결과를 아래에 누적(최근 30줄 우선 표시)
- “텍스트 속도” 옵션: 즉시/느리게(타이핑)

## 3. 입력 모델
- 기본: 버튼 선택
- 옵션(후속): 커맨드 입력창(텍스트) → Command로 파싱

## 4. 에러/가이드
- 유효하지 않은 입력은 무시하고 SystemLog로 안내
- 저장 실패 시 토스트 + 마지막 정상 저장으로 복구

## 5. 접근성
- 폰트 크기 옵션
- 대비(다크 모드) 지원
